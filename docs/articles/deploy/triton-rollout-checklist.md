# Triton 上线检查清单

**日期**: 2026-03-28
**分类**: 深度学习部署
**标签**: Triton, DL_Deploy, 推理服务

---

## 为什么需要清单

模型能够在本地跑通，并不代表它已经准备好上线。真正到了服务阶段，输入输出协议、显存水位、批处理策略和报警规则都要一起考虑。

## 上线前建议确认

1. **输入输出约束**: 明确张量 shape、dtype 和归一化方式。
2. **资源限制**: 提前测出显存峰值和 batch 上限，不要把安全边界留给线上。
3. **回退策略**: 新模型上线前，保留旧版本回切入口。
4. **观测指标**: 至少准备延迟、吞吐、失败率和显存占用四项。

## 一个实用习惯

把这份清单写成发布说明的一部分，比只存在脑子里更可靠。
